# Контрольні запитання
1.Що таке generics? Які їхні переваги?
Generics — це узагальнені типи, що дозволяють створювати класи, методи й колекції, які працюють із будь-яким типом даних без втрати типобезпеки.
Переваги: безпечність типів, відсутність кастів (перетворень типів), краща продуктивність, універсальність коду.

2.Які основні відмінності між ArrayList і List<T>?
ArrayList зберігає об’єкти типу object (без типобезпеки).
List<T> — узагальнений список, що працює з певним типом (List<int>), без потреби перетворень.

3.Чим відрізняється Dictionary<TKey,TValue> від List<T>?
Dictionary<TKey, TValue> зберігає пари "ключ–значення", забезпечує швидкий доступ за ключем.
List<T> — просто список елементів, доступ за індексом.

4.У чому перевага LINQ над класичними циклами?
Дозволяє записувати запити до колекцій коротше, зрозуміліше і декларативно, без ручних циклів.

5.Як працює ключове слово finally?
Блок, що виконується завжди після try / catch — навіть якщо сталася помилка. Використовується для очищення ресурсів.

6.Коли доцільно створювати власні класи винятків?
Створюють, коли потрібно описати специфічну помилку у власній програмі (для точнішої діагностики чи обробки).
